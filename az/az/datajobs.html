Məlumat Emalı və Mənbə İşləri

Bu səhifə layihənin məlumat toplama, yoxlama və emal mərhələsini sənədləşdirir. Məqsəd yalnız neçə fayl endirildiyini göstərmək deyil, hər mənbənin hansı seriya ailəsini verdiyini, hansı boşluğun bağlandığını, hansı seriyanın isə artıq açıq mənbədən endirilə bilməyib hesablanmalı olduğunu aydın göstərməkdir.

İcra xülasəsi

Toplama mərhələsi əvvəlki “məlumat yoxdur” qiymətləndirməsini əhəmiyyətli dərəcədə dəyişdi. SSC/Azstat, AMB, Dövlət Gömrük Komitəsi, Maliyyə Nazirliyi, SOFAZ, IMF, FRED, ECB, WRDS/CRSP və Refinitiv/LSEG mənbələri üzrə əsas makro, xarici sektor, monetar, fiskal, bank və suveren-risk seriyaları əldə edilib və ayrıca manifestə salınıb. FRED üzrə əvvəlki üç uğursuz sətir də API vasitəsilə bağlanıb: gündəlik DFF, ECBDFR və ECBMRRFR artıq raw/fred/ qovluğundadır.

Nəticə budur: 11.1.1 və 11.1.2 üzrə açıq-mənbəli məlumat bloku praktik olaraq tamamlanıb. Qalan iki məsələ məlumat toplama işi deyil, emal işidir: idxal qiymət indeksi açıq rəsmi seriya kimi tapılmadığı üçün vahid-dəyər indeksi kimi qurulmalıdır; rəsmi baza inflyasiya isə AMB sənədlərində nöqtə oxunuşları kimi çıxır, lakin maşın-oxunan uzun zaman seriyası kimi verilmir. Buna görə davamlı aylıq core CPI seriyası komponentlərdən qurulmalı və AMB-nin rəsmi nöqtə oxunuşları ilə kalibrlənməlidir.

Şəkil 3A.1. Toplanmış raw faylların əsas mənbə qrupları üzrə bölgüsü. SSC/Azstat çox saylı sektoral və məhsul cədvəllərinə görə ən böyük hissəni tutur.
Şəkil 3A.2. Məlumat işlərinin statusu. Yaşıl bloklar modeldə birbaşa istifadə oluna bilən seriyalardır; sarı bloklar açıq məlumatdan qurulacaq törəmə/proksi seriyalardır.

Bağlanmış məlumat boşluqları

Bağlandı

FRED policy-rate seriyaları

DFF, ECBDFR və ECBMRRFR əvvəlcə fredgraph endpointində bloklanmışdı. FRED API vasitəsilə CSV və JSON kimi endirildi; manifestdə uğursuz sətir qalmadı.

Bağlandı

Suveren spread

WRDS identifikator və proxy verdi; əsas suveren yield tarixi Refinitiv/LSEG-dən alındı. Əsas spread indi Azərbaycan YTM minus ABŞ Treasury CMT kimi 2026-06-ya qədər hesablanır.

Aydınlaşdırıldı

Nazirliyin model paketi

Sektoral iqtisadiyyat, büdcə və xarici ticarət modelləri iş kitabları və metodologiya sənədlərində verilmişdir. Bank sektoru davranış modeli isə ayrıca qurulmalıdır.

Qurulmalı olan iki törəmə seriya

İki seriya açıq mənbədə rəsmi, uzun, maşın-oxunan zaman seriyası kimi tapılmadı. Bu, işin dayanması demək deyil: hər ikisi artıq əldə olan məlumatdan metodoloji olaraq qurula bilər və hesabatda ayrıca iş kimi qeyd edilməlidir.

$$ UV_t = \frac{\sum_i V_{i,t}}{\sum_i Q_{i,t}},\qquad \pi^{m}_t = \Delta \log(UV_t) $$(D1)

İdxal qiymət proksisi. Rəsmi import-price index / unit-value index açıq seriya kimi təsdiqlənmədi. Ona görə idxal qiymət kanalı iki yolla qurulacaq: birincisi, Gömrük və SSC ticarət cədvəllərindən dəyər/həcm nisbəti ilə vahid-dəyər indeksi; ikincisi, tərəfdaş CPI/PPI və məzənnə əsasında trade-weighted xarici qiymət göstəricisi. Bu, Nazirliyin CPI tənliyindəki yanaşmaya uyğundur: idxal inflyasiyası ayrıca rəsmi indeksdən çox NEER, dünya ərzaq qiyməti, neft və tərəfdaş qiymətləri ilə təmsil olunur.

$$ \pi^{core}_t = \pi_t - \pi^{admin}_t - \pi^{seasonal}_t \quad \text{(AMB tərifinə yaxınlaşdırılmış göstərici)} $$(D2)

Core / baza inflyasiya. AMB baza inflyasiyanı hesablayır və rəsmi sənədlərdə nöqtə oxunuşları kimi dərc edir; lakin ayrıca CSV/XLSX uzun seriya kimi vermir. Tərif vacibdir: AMB-nin baza inflyasiyası food-energy deyil, tənzimlənən və mövsümi mallar/xidmətlər çıxılmaqla hesablanır. Buna görə davamlı aylıq göstərici SSC komponentlərindən rekonstruksiya edilməli və AMB-nin rəsmi nöqtə oxunuşları ilə validasiya edilməlidir.

Emal pipeline-ı

Məlumat işi üç səviyyəyə bölünür. Birinci səviyyə mənbə fayllarının qorunmasıdır: raw fayllar dəyişdirilmir və manifestdə saxlanılır. İkinci səviyyə standartlaşdırmadır: tarix, valyuta, tezlik, adlandırma və mənbə metadata-sı vahidləşdirilir. Üçüncü səviyyə model seriyalarının qurulmasıdır: spread, import-price proxy və core CPI kimi törəmə dəyişənlər ayrıca hesablanır və bu hesablamalar təkrar istehsal oluna bilən notebook / script ilə sənədləşdirilir.

Nəticə
Açıq və lisenziyalı mənbələr üzrə məlumat toplama artıq ümumi bloklayıcı deyil. Qalan iş məlumat toplamaqdan daha çox iki törəmə seriyanı qurmaq, üç təhvil verilmiş model blokunu pipeline-a keçirmək və bank sektoru davranış modelini CBAR/IMF FSI məlumatı ilə qurmaqdır.
