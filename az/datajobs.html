Data Processing and Provenance

This page records how the whole forecasting evidence base is organised and converted into model-ready series. It covers the Ministry's legal and methodological documents, the CAEM workbook, the macroeconometric workbooks, public institutional data, licensed market data, derived variables and the reproducible outputs used in the model pages. It is a full-project provenance page for the MİİS forecasting system.

Purpose

The task is not only to collect macroeconomic data. The task is to make every forecast, scenario and statutory table traceable from a legal or methodological requirement to a specific source series, transformation and model equation. This page therefore sits between the data foundation and the econometric model results: it explains what the system uses, why each source is needed, and what remains as a construction or internal-data issue.

Derived variables

Several important variables are model inputs rather than simple downloads. They must be shown as formulas, with the source files and validation checks documented.

$$ spread_t = YTM^{AZE}_t - YTM^{UST, matched}_t $$(D1)

Sovereign risk premium. The primary series is built from Refinitiv/LSEG Azerbaijan sovereign Eurobond yield-to-maturity data for XS1678623064 and XS1678623734, matched to US Treasury constant-maturity benchmarks. WRDS/CRSP data are used for benchmark validation and robustness.

$$ UV_t = \frac{\sum_i V_{i,t}}{\sum_i Q_{i,t}},\qquad \pi^m_t = \Delta\log(UV_t) $$(D2)

Import-price proxy. A clean official import-price index was not confirmed as a public long series. The appropriate treatment is to construct a unit-value index from customs and trade value/quantity data, and cross-check it against partner-price and exchange-rate indicators.

$$ \pi^{core}_t \approx \pi_t - \pi^{administered}_t - \pi^{seasonal}_t $$(D3)

Core/base inflation. CBAR publishes official core inflation as point readings in monetary-policy material rather than as a long machine-readable series. A continuous monthly proxy must be reconstructed from CPI components and reported with a definition caveat because CBAR's base inflation excludes administered and seasonal items.

What this changes for the work plan

Proceed

11.1.1 public-data model

The public and licensed evidence base is sufficient for the statutory macro forecast, scenario block, external monetary conditions and sovereign-risk channel.

Proceed

11.1.2 model package

Three major Ministry model blocks are present with equations and workbooks: sectoral economy, budget and foreign trade. They should be coded and validated, not re-requested.

Build

Banking and derived-price modules

The banking behavioural module, import-price proxy and core-inflation proxy are build tasks using data already held. They should be flagged as transformations, not as missing public downloads.

Governance rule

Every model input should carry five fields: original source, native frequency, transformation, model owner and vintage. This is the acceptance test for the data layer. If a series is derived, the formula must be stored beside the output; if a source is internal, the page should say so without weakening the public-data prototype.

Conclusion
The evidence base now covers the Ministry's own model assets, the CAEM system, public institutional sources, licensed market data and produced econometric outputs. The remaining work is not a general data shortage: it is controlled transformation, model coding and the banking-module build.
